Building Automations Without Code

You do not need to understand code to build automations. You need to understand your own workflows well enough to describe what should happen, in what order, and when.

Overview

Automation is not about robots or science fiction. It is about the report you build every Monday morning. The 14 emails you send after every client meeting with slightly different details. The data you copy from one spreadsheet to another every week because nobody ever built a better system.

This module teaches you to identify the workflows that eat your time and to build automations for them using Claude as your builder. You will not learn to code. You will learn to describe what you need precisely enough that Claude can build it for you, and you will learn to connect the tools you already use so they talk to each other without your manual intervention.

By the end, you will have at least one working automation that saves you 30 or more minutes per week. That is 26 hours a year. From one automation.

What Automation Actually Means

Forget the word's baggage. In practice, automation means: something that used to require your manual effort now happens without it, or with significantly less of it.

There are three levels:

  1. Scripts: A file that runs a sequence of steps when you tell it to. "Process this week's data" — you click run, it executes, done. You built it once with Claude's help and now you just trigger it.
  2. Scheduled automations: Scripts that run on a timer. Every Monday at 7am, the weekly report generates itself. Every evening, new files get sorted into the right folders. You do not click anything.
  3. Triggered automations: Workflows that fire when something happens. A new email arrives and gets automatically categorized. A file lands in a folder and gets processed. A form submission generates a personalized response.

Most practitioners should start at level 1 and work up. A script you run manually is simpler to build, easier to trust, and still saves massive time compared to doing the work by hand.

Using Claude to Create Scripts

Here is the core practitioner automation workflow:

  1. Describe the task to Claude in plain language, with enough detail that a new employee could follow the instructions.
  2. Ask Claude to build it. In Claude.ai, ask for a script. In Claude Code, ask it to create and save the script directly to your machine.
  3. Test it on a small sample of real data. Not the full batch — a subset.
  4. Refine it by telling Claude what went wrong or what needs to change.
  5. Save it and run it whenever you need it.

You do not need to understand every line of the script. You need to understand what it does, what it expects as input, and what it produces as output. This is the same standard you apply to any tool. You do not understand how your dishwasher works at the engineering level, but you know what goes in, what comes out, and what to check if something seems off.

The description is the hard part. Most automation failures happen because the human described the task too vaguely. "Process my invoices" fails. "Read every PDF in the invoices/ folder, extract the invoice number, date, vendor name, and total amount, and create a CSV file with one row per invoice sorted by date" succeeds. The precision of your description determines the quality of the automation.

What makes a good automation candidate?

Not every task should be automated. The best candidates share these traits:

  • Repetitive. You do it at least weekly, ideally daily.
  • Rule-based. The steps are consistent even if the data changes. "If X, do Y" — not "use your best judgment."
  • Time-consuming. It takes more than 10 minutes each time. Automating a 2-minute task is not worth the setup effort.
  • Low-stakes for errors. Start with tasks where a mistake is easy to catch and fix, not tasks where an error costs $50K.

Common high-value candidates: weekly report generation, data entry from one format to another, file organization, email template population, invoice processing, meeting notes formatting, client data updates.

Connecting Claude to Your Existing Tools

Scripts that run on your machine are powerful. But the real leverage comes from connecting Claude's intelligence to the tools you already use — your email, your CRM, your project management system, your spreadsheets.

Zapier and Make

These are visual automation platforms. You build workflows by connecting triggers and actions from different apps. No code required — you click and configure.

How Claude fits in: Both Zapier and Make have Claude/Anthropic integrations. This means you can build workflows like:

  • New email arrives with attachment -> Claude analyzes the attachment -> Routes to correct team member based on content
  • Form submission received -> Claude generates a personalized response -> Email sent automatically
  • New row added in Google Sheet -> Claude generates a summary -> Posts to Slack channel
  • Calendar event ends -> Claude drafts follow-up email based on meeting title and attendees -> Draft saved for your review

The pattern is always the same: something happens (trigger), Claude processes it (intelligence), and something else happens as a result (action). You are inserting Claude's language understanding into the middle of workflows that already exist.

n8n: The self-hosted option

n8n is like Zapier but you can run it on your own machine, and the free tier is more generous. If you are comfortable with what you learned in Module 2 about terminals and Claude Code, n8n is worth exploring. Claude Code can help you set it up. The key advantage: no per-task pricing. You build as many automations as you want without incremental cost.

Start with Zapier. It has the most integrations, the simplest interface, and a free tier that handles basic automations. Once you outgrow it — and you will if you take this seriously — migrate to Make or n8n for more power and lower cost. Do not over-optimize your tool choice before you have built your first automation.

Practical Automation Patterns

Email templates that think

Most email templates are static — fill in the blanks, send. With Claude in the loop, your templates become dynamic. Instead of "Dear [NAME], Thank you for your [THING]," you feed Claude the context and get a genuinely tailored response every time.

Build this in Zapier: when you add a row to a specific Google Sheet (client name, context, purpose), Claude generates a complete email draft in your preferred tone, and it appears as a Gmail draft ready for your review and send.

Report generation

If you produce any recurring report — weekly status updates, monthly analytics summaries, quarterly reviews — you can automate the draft. Feed Claude the raw data from a spreadsheet, a database export, or a dashboard screenshot and get a formatted narrative report. You review, edit where needed, and send. The 90-minute report becomes a 15-minute review.

Data transformation pipelines

Data almost never arrives in the format you need. Clients send spreadsheets with different column names. Vendors use different date formats. Systems export data that needs to be cleaned before it is useful. Build a Claude Code script that accepts messy input and produces clean output. Run it every time you get new data. The hour of reformatting becomes a 10-second script execution.

Content repurposing

You wrote a blog post and now you need a LinkedIn version, three tweets, an email newsletter paragraph, and a slide deck summary. Or your company published a case study and you need a client-facing one-pager. Claude can transform one piece of content into multiple formats with consistent messaging. Build the prompt template once, reuse it every time you publish.

Building Your Personal Automation Stack

Think of your automation stack as three layers:

  1. Layer 1 — Claude Code scripts. Local scripts for file processing, data transformation, and batch operations. These run when you tell them to. You built them in Module 2 and now you are expanding the collection.
  2. Layer 2 — Cloud workflows. Zapier/Make automations connecting your SaaS tools with Claude intelligence in the middle. These run on triggers or schedules automatically.
  3. Layer 3 — Claude.ai Projects. Your thinking environment from Module 1, used for the tasks that still need human judgment but benefit from structured AI assistance.

Each layer handles a different kind of work. Scripts handle bulk operations. Cloud workflows handle recurring triggers. Projects handle judgment calls. Together, they form a system that handles a significant portion of your routine work with minimal ongoing effort from you.

Start small. One script, one cloud workflow, one well-configured Project. Prove the value. Then expand.

Exercise

Create an automation that saves you 30+ minutes per week on a real recurring task.

This is the most important exercise in the Practitioner path. It is where the theory becomes a permanent change in how you work.

  1. Audit your week. List every task you did this week that was repetitive, rule-based, and took more than 10 minutes. Be specific — not "email" but "draft follow-up emails after client meetings using notes from the meeting."
  2. Pick one task that you can describe precisely enough for someone else to do it. If you cannot describe it precisely, it is not ready to automate.
  3. Choose your tool. If the task involves local files, use Claude Code to build a script. If the task involves connecting cloud services, use Zapier or Make. If you are unsure, ask Claude which approach fits better.
  4. Build the automation. Describe the task to Claude in detail. Test on real data. Refine until the output is correct and reliable.
  5. Run it for real at least twice on actual work tasks. Measure the time savings.
  6. Document it. Write a one-paragraph description of what the automation does, what it needs as input, and where the output goes. Future you will thank present you.

The bar for success: next week, when this task comes up again, you use your automation instead of doing it manually. If you built something you will not actually use, go back and build something you will.
