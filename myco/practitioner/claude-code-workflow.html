Claude Code as a Workflow Tool

You do not need to be a developer to use Claude Code. You need to be someone with files, data, and repetitive tasks who is tired of doing them by hand.

Overview

Claude Code is a command-line tool that lets Claude read files on your computer, write files, run commands, and interact with your actual work environment. Most people assume it is for programmers. It is not. It is for anyone who works with files and data and wants an agent that can act on them directly.

If you have ever spent an afternoon renaming 200 files, copying data between spreadsheets, converting document formats, or reorganizing a folder structure, Claude Code can do that in minutes. Not because it writes code for you (though it does), but because it executes tasks directly on your machine, working with your real files in real time.

This module teaches you how to use Claude Code as a workflow tool, even if you have never opened a terminal before. You will learn enough to be dangerous, and you will process a batch of real files to prove it.

What Claude Code Is and Why It Matters

Claude.ai lives in a browser tab. It can read what you paste or upload, and it can generate text and artifacts. But it cannot touch your files. It cannot open a spreadsheet on your desktop, rename your downloads folder, or process a batch of invoices sitting in your Documents directory.

Claude Code can. It runs in your terminal and has direct access to your file system. When you tell Claude Code to "rename every file in this folder from the format 'Invoice_2024_001' to '2024-01-Invoice'," it does not give you instructions on how to do it. It does it.

This is a fundamentally different relationship with AI. You go from "Claude advises me" to "Claude works for me." The distinction matters more than any feature list.

Access note: Claude Code requires a paid API plan or a Claude Pro/Max subscription. This module assumes you have access. If you do not yet, read through the material so you understand the capability, and set up access before the exercise.

The Terminal: What You Need to Know

You do not need to become a terminal expert. You need to know five things:

  1. How to open it. On Mac: search "Terminal" in Spotlight. On Windows: install Windows Terminal from the Microsoft Store. On Linux: you probably already know.
  2. Where you are. The terminal always has a "current directory" — the folder you are working in. Type pwd (Mac/Linux) or cd (Windows) to see it.
  3. How to move. cd Documents moves you into the Documents folder. cd .. moves you up one level. cd ~/Desktop jumps straight to your Desktop.
  4. How to look around. ls (Mac/Linux) or dir (Windows) lists what is in the current folder.
  5. How to start Claude Code. Type claude and press Enter. You are now talking to Claude with access to your files.

That is the entire terminal literacy you need for this module. Five concepts. Everything else, you tell Claude Code what you want in plain English and it handles the technical details.

Installing Claude Code

Claude Code installs through npm (Node Package Manager). If that sentence means nothing to you, here is what to do:

  1. Install Node.js from nodejs.org. Download the LTS version. Run the installer. Accept defaults.
  2. Open your terminal and type: npm install -g @anthropic-ai/claude-code
  3. Type claude to start. It will walk you through authentication on first run.

If any step fails, paste the error message into Claude.ai and ask for help. This is a real skill — using Claude to troubleshoot Claude.

Using Claude Code on Real Work

File management at scale

This is where most non-technical people first feel the power. Navigate to a folder with files you need to organize, start Claude Code, and describe what you want:

  • "Rename every PDF in this folder to include today's date at the beginning of the filename."
  • "Sort these 150 photos into folders by year and month based on their file creation date."
  • "Find every .docx file in my Documents folder that mentions 'Project Alpha' and copy them to a new folder called 'Alpha Archive'."
  • "Convert all these .xlsx files to .csv format."

Claude Code reads your request, writes a script to accomplish it, shows you what it plans to do, and asks for confirmation before executing. You are in control at every step. If something looks wrong, say no and clarify what you meant.

Data processing

This is the use case that saves the most time. If you work with spreadsheets, CSVs, or any structured data:

  • "Read sales_q4.csv and create a summary showing total revenue by region, average deal size, and the top 5 accounts by revenue."
  • "Merge these three CSV files, remove duplicate rows based on email address, and sort by last name."
  • "Read this spreadsheet of property listings and create a new file that only includes listings over $500K with 3+ bedrooms, sorted by price per square foot."
  • "Compare last month's inventory file to this month's and show me what changed — new items, removed items, price changes."

You describe the transformation in plain language. Claude Code writes the code, runs it, and produces the output file. You never need to understand the code itself. You need to understand whether the output is correct.

Research and document processing

Claude Code can read and process documents in bulk in ways that Claude.ai cannot, because it works through files sequentially on your machine without upload limits:

  • "Read every PDF in this folder and create a spreadsheet with the contract name, start date, end date, and total value from each one."
  • "Scan all the text files in this directory and find every mention of a dollar amount. Create a report."
  • "Read these 20 meeting notes and create a single document listing every action item with the person responsible and the date mentioned."
The verification habit: Claude Code is powerful but not infallible. Always spot-check the output. Open the resulting files and verify a few data points against the originals. Trust but verify — especially the first time you run a new kind of task.

When to Use Claude.ai vs Claude Code

This is a decision you will make daily once you are fluent with both tools. Here is the framework:

Use Claude.ai when:

  • You need conversation, brainstorming, or back-and-forth refinement
  • You are working with a small number of files you can upload manually
  • You want artifacts — polished documents, diagrams, formatted output
  • You are on your phone or a machine without Claude Code installed
  • The task is primarily about thinking, not doing

Use Claude Code when:

  • You need to process many files at once
  • You need to read from or write to your local file system
  • The task involves transformation — renaming, reformatting, merging, splitting
  • You need to automate something that happens regularly
  • You want Claude to execute, not just advise

The simplest rule: if you want a conversation, use Claude.ai. If you want an action, use Claude Code. Many real workflows use both — think through your approach in Claude.ai, then switch to Claude Code to execute it at scale.

Practical Patterns

Three patterns cover about 80% of the file work most knowledge workers do:

The extraction pattern

You have a collection of documents and you need specific information pulled out of each one into a single summary. Contracts, invoices, reports, applications — any batch of documents with a consistent structure. Navigate to the folder, start Claude Code, and say "Read every [document type] in this folder and extract [specific fields]. Create a spreadsheet with one row per document."

The transformation pattern

You have data in one format and need it in another. CSV to formatted report. Raw data to client-ready summary. One naming convention to another. Describe the input and the desired output, and Claude Code handles the conversion.

The comparison pattern

You have two versions of something and need to understand what changed. Last quarter versus this quarter. Old vendor list versus new. Previous draft versus current. "Compare these two files and create a report showing every difference, categorized by type of change."

Exercise

Use Claude Code to process a batch of files that would take you an hour manually.

This has to be real. Gather actual files from your work — not sample data, not tutorials. The goal is to experience the time savings firsthand.

  1. Install Claude Code if you have not already. Verify it runs by typing claude in your terminal.
  2. Choose a batch task from your actual work. Good candidates: a folder of files that need renaming, a set of CSVs that need merging, a collection of documents that need specific data extracted, or a directory that needs reorganizing.
  3. Navigate to the folder with your files. Start Claude Code.
  4. Describe what you want in plain English. Be specific about the input (which files, which folder) and the desired output (what the result should look like).
  5. Review Claude Code's plan before it executes. If anything looks wrong, say so and refine.
  6. After execution, verify the output. Spot-check at least 5 results against the originals.
  7. Note how long the process took versus how long it would have taken manually.

Success criteria: you processed a real batch of files, the output was correct, and the time savings were significant. If the task took Claude Code 3 minutes and would have taken you 60, you understand the value proposition.
