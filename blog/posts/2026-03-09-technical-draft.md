**Title: Configuring NVIDIA GPU Passthrough in NixOS 24.05**

**Problem Statement**  
After upgrading to NixOS 24.05, users may encounter GPU detection failures when using NVIDIA cards in a virtualized or passthrough setup. This often stems from incomplete PCI device configuration or missing kernel modules.  

**Solution: PCI Device Configuration & Driver Installation**  

1. **Identify GPU PCI ID**  
   Run:  
   ```bash  
   lspci -v | grep -i nvidia  
   ```  
   Example output:  
   ```  
   01:00.0 VGA compatible controller: NVIDIA Corporation GK110B [GeForce GTX 680] (rev a1)  
   ```  
   Note the `01:00.0` slot.  

2. **Update `hardware-configuration.nix`**  
   Add the following block to your config:  
   ```nix  
   pciDevices = [  
     {  
       vendor_id = "10de";  
       device_id = "1180";  
       bus = "01";  
       slot = "00";  
       function = "0";  
       driver = "nvidia";  
     }  
   ];  
   ```  
   Replace `device_id` with your GPU’s ID (e.g., `1180` for GTX 680).  

3. **Install NVIDIA Driver**  
   Run:  
   ```bash  
   nix-env -iA nixos.nvidia-drivers-535  
   ```  
   Replace `535` with your GPU’s compatible driver version (check NVIDIA’s website).  

4. **Rebuild System**  
   ```bash  
   nixos-rebuild switch --upgrade  
   ```  

**Working Configuration Example**  
```nix  
{ config, lib, ... } :=  
{  
  hardware-configuration = {  
    pciDevices = [  
      { vendor_id = "10de"; device_id = "1180"; bus = "01"; slot = "00"; function = "0"; driver = "nvidia"; }  
    ];  
  };  

  services.xserver = {  
    enable = true;  
    driver = "nvidia";  
    additionalOptions = {  
      "NvidiaXServerSettings" = "Device=01:00.0";  
    };  
  };  
};  
```  

**Troubleshooting**  
- **GPU Not Detected**:  
  Check `dmesg | grep -i nvidia` for errors. Ensure the PCI ID in `hardware-configuration.nix` matches `lspci`.  
- **Driver Failures**:  
  Verify the driver version matches your GPU (e.g., `nvidia-smi` after installation).  
- **Kernel Module Missing**:  
  Run `lsmod | grep nvidia` to confirm modules are loaded. If not, reinstall the driver.  
- **Passthrough Issues**:  
  Ensure BIOS/UEFI enables PCIe passthrough and disables secure boot. Use `intel-iommu=on` in kernel parameters if using VT-d.  

**Final Checks**  
After reboot, run:  
```bash  
nvidia-smi  
```  
A working GPU will show metrics. Use `nvidia-xserver-config` to validate Xorg settings.  

This setup ensures stable NVIDIA GPU operation in NixOS, critical for AI workloads requiring CUDA acceleration.
