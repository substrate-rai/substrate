## Troubleshooting

**"The option `services.ollama.acceleration` does not exist"** — You're on NixOS unstable. Replace `acceleration = "cuda"` with `package = pkgs.ollama-cuda` in your Ollama config. Rebuild with `sudo nixos-rebuild switch --flake .#substrate`.

**"connection refused on localhost:11434"** — Ollama service isn't running. Check `systemctl status ollama`. Common after kernel updates — rebuild and restart.

**"no NVIDIA GPU detected"** — Missing `allowUnfree = true` or `hardware.graphics.enable = true` in your NixOS config. Both are required for NVIDIA drivers.

**CUDA process not showing in nvidia-smi** — You're using `pkgs.ollama` (CPU-only) instead of `pkgs.ollama-cuda`. Check your config.

**Slow first request** — Ollama loads the model into VRAM on first use. Subsequent requests are fast. Keep the service running to avoid reload delay.

EXAMPLE — a good fix section:

## Error: `services.ollama.acceleration` Does Not Exist

After upgrading to NixOS unstable (nixpkgs 2026-03-04, version 26.05):

```
$ sudo nixos-rebuild switch --flake .#substrate
error: The option `services.ollama.acceleration` does not exist.
```

### Fix

Replace the `acceleration` option with the CUDA package:

```nix
services.ollama = {
  enable = true;
  package = pkgs.ollama-cuda;
};
```

Rebuild:

```bash
sudo nixos-rebuild switch --flake .#substrate
```
