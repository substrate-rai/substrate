## AI on Linux in 2026: What Actually Works

This is a practical survey based on what we actually run in production at Substrate — a 30-agent AI system on a single laptop with an NVIDIA RTX 4060 (8GB VRAM), 62GB RAM, and NixOS.

---

## Hardware

| Component | Our Setup | Minimum Viable |
|-----------|-----------|----------------|
| CPU | AMD Ryzen 7 (16 threads) | Any modern 8-core |
| GPU | NVIDIA RTX 4060 (8GB VRAM) | 6GB+ VRAM for 7B models |
| RAM | 62GB DDR5 | 16GB minimum |
| Storage | 1.8TB NVMe | 256GB+ NVMe |

AMD GPUs work with ROCm but NVIDIA + CUDA remains the path of least resistance for local inference in 2026.

---

## What We Actually Run

### Local Inference: Ollama

Ollama is the simplest way to run open models locally. On NixOS:

```nix
services.ollama = {
  enable = true;
  acceleration = "cuda";
};
```

We run **Qwen3 8B** as our primary local model. Performance on RTX 4060:
- ~40 tokens/sec generation
- ~5.5GB VRAM usage
- Handles: drafting, summarization, health checks, creative writing

### Image Generation: Stable Diffusion (ComfyUI)

SDXL Turbo runs well on 8GB VRAM with these settings:
- Steps: 6
- CFG: 1.0
- Resolution: 512x512 portraits, 1024x512 scenes

### Cloud Inference: Claude API

For complex reasoning, code review, and architecture work, we route to Claude via the Anthropic API. Cost: ~$0.40/week with our two-brain routing system.

## Tool Comparison

| Tool | Use Case | VRAM | Setup Difficulty |
|------|----------|------|-----------------|
| Ollama | Chat, generation, agents | 5-8GB | Easy |
| llama.cpp | Embedded, custom builds | 4-8GB | Medium |
| vLLM | High-throughput serving | 8GB+ | Medium |
| ComfyUI | Image generation | 6-8GB | Medium |
| text-generation-webui | Interactive chat UI | 6-8GB | Easy |

### What works well

- **Ollama** — dead simple, systemd integration, CUDA auto-detection
- **llama.cpp** — best performance per watt, GGUF format is the standard
- **ComfyUI** — node-based workflows for image generation

### What's improved since 2025

- Quantized models (Q4_K_M, Q5_K_M) run well on consumer GPUs
- Ollama's API is now stable and well-documented
- NixOS has first-class CUDA support via `services.ollama.acceleration`

## NixOS Configuration

Our complete development environment in a flake:

```nix
services.ollama = {
  enable = true;
  acceleration = "cuda";
};

nixpkgs.config.allowUnfree = true;
hardware.graphics.enable = true;

environment.systemPackages = with pkgs; [
  python311
  git curl wget htop
  nvtopPackages.full
];
```

## Substrate Note

We run 30 AI agents on a single RTX 4060 laptop. The trick is tier scheduling — only one agent uses the GPU at a time, with circuit breakers and queue management. 95% of inference runs locally via Ollama. The remaining 5% goes to Claude API for tasks that need stronger reasoning.

Total monthly cost: under $2 in cloud API fees. Everything else is free.

## Troubleshooting

| Error | Fix |
|-------|-----|
| `CUDA driver not found` | Set `hardware.graphics.enable = true` and `nixpkgs.config.allowUnfree = true` |
| `Ollama not found` | Enable `services.ollama.enable = true` in NixOS config |
| `Out of VRAM` | Use a smaller quantization (Q4_K_M) or smaller model |
| GPU not detected | Run `nvidia-smi` — if it fails, check NVIDIA driver is loaded |
